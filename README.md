## Структура проекта

Из сторонних библиотек использовались vue-select и date-picker
Вынес в отдельные компоненты поля (input, textarea, date-picker, select)

Валидация полей реализована через composable useValidation. По функционалу получился обрезанный абстрактный аналог vuelidate

Основной компонент проекта - order-detail, влючает в себя компоненты: форму редактирование и карточку с отображением заказа

## Принцип работы
При любых изменениях полей создается объект draft (черновик), куда добавляются несохраненные данные и отправляются в localStorage. Если страница обновляется, данные при наличии подтягиваются из черновика. При сохранении изменений или отмене драфт удаляется и обновляются основные данные.

На перспективу пользователь может и просто по сторонней ссылке из шапки кликнуть, в таком случае можно показать пользователю уведомление, что введенные данные будут утеряны перед уходом с текущей страницы.

Управление режимом осуществляется через get параметр ?edit=true.
